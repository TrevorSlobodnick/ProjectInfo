Pokemon Card Collection

Project Info

Description

This is an app that I made for myself to help manage/track my pokemon card collection.

My Motivation

After watching a lot of youtubers open really old, expensive pokemon card packs, I wanted to get back into it myself. I was surprised at what cards were considered "high value", because a lot of the cards that looked cool weren't worth nearly as much as some of these cards. Although I got rid of most of my collection when I was a kid, I still kept some, and I realized I had no way of knowing if I had one of these high value cards, because I always thought the best cards had the highest hp... So I created this app to track all my cards

Technologies

  • React
  • PHP

Challenges

Creating A Custom <select>
I always knew <select>'s can only have <option>'s as children, but I thought that <option>'s could have whatever they wanted as children. This is not true. The content of an option tag can only be a string or number, but I wanted to add an image and string for each option. So I had to find a way to create a custom drop-down list to display each card set. Luckily React has a library called react-select that is very customizable and looks great, so I decided to go with that.
Status Updates/Notifications
I wanted a way to let the user know that after taking some sort of action on the website, that somthing is actually happening. I mainly wanted this functionality for when a user, or I, add a card, and if the card is successfully added, show a quick popup message, and if it failed, show an error message. I found a React library called react-toastify, which does exactly what I was looking for. It allows me to create a short popup that appears for a specified length of time and then it will disappear off the screen.

Improvements

Easily Add Card
Currently, You have to enter the card number and match a set icon from the drop-down list to the one on the bottom of the card in order to add a card to the collection. Entering a number is no problem, but since there are a lot of sets, and the sets list is only getting larger, another solution would be ideal. If each card had some sort of serial number so it could be easily identified, that would be great. But since I can't change the way cards are made, the only other solution I can think of would be to implement some sort of AI that can accurately identify a pokemon card, but since this is out of the scope of the project, it will not be implemented.
